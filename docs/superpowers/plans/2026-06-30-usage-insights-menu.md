# Usage Insights Menu Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and deploy the compact native-shell Coco menu with full 30-day usage submenus, model-aware estimated cost, copy summary, and branded PNG snapshots.

**Architecture:** Keep the app dependency-light and local-only. Extend `TokenWindow`/`UsageSample` with aggregate detail fields, replace usage log readers with file-aware parsers where detail metadata is needed, and render native `NSMenu` submenus for 30-day details/actions. Generate share snapshots with AppKit drawing.

**Tech Stack:** Swift 6, AppKit, Foundation, Sparkle, GitHub Releases helper scripts.

---

### Task 1: Add Usage Detail Data

**Files:**
- Modify: `Sources/CocoUsageBar/main.swift`

- [ ] **Step 1: Extend aggregate structs**

Add detail fields to `TokenWindow`: session count, source tokens, model tokens, day tokens, pricing basis, and unpriced/fallback token counters. Add metadata fields to `UsageSample`: session ID, source, model, pricing basis, and fallback flag.

- [ ] **Step 2: Keep snapshot privacy**

Ensure any internal session ID set is not encoded to `snapshot.json`. Encode only counts and aggregate dictionaries.

- [ ] **Step 3: Build**

Run: `swift build -c release`

Expected: build succeeds.
